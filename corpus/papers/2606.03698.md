# Multi2: Hierarchical Multi-Agent Decision-Making with LLM-Based Agents in Interactive Environments

Sangeun Park    Minhae Kwon

###### Abstract

A central goal of large language model (LLM) research is to build agentic systems that can plan, act, and adapt through sustained interaction with dynamic environments. While recent LLM-based agents exhibit impressive contextual reasoning, their long-horizon decision-making remains fragile, often suffering from objective drift, where goals and plans drift over extended interactions. We introduce Multi2, a hierarchical multi-agent decision-making framework that explicitly decomposes agent behavior into complementary roles. A high-level agent (System 1) focuses on context-aware sub-goal generation using supervised fine-tuning (SFT), while a low-level agent (System 2) executes atomic actions through offline-to-online reinforcement learning (RL) in interactive environments. This separation enables stable long-horizon control, mitigates objective drift, and allows efficient adaptation. Across diverse interactive environments, Multi2 consistently outperforms strong agentic baselines, demonstrating improved robustness and coordination in multi-turn interaction. Beyond performance, we introduce and release three hierarchical benchmark datasets, filling a long-standing gap in training and evaluating hierarchical decision-making for LLM-based agents.

## 1 Introduction

A central goal in agentic artificial intelligence (AI) is to build LLM-based agents that can solve complex tasks through closed-loop interaction with dynamic environments, where each action changes the world state and must be followed by decisions conditioned on new observations (Dong et al., [2025](#bib.bib3 "Enhancing decision-making of large language models via actor-critic"); Wu et al., [2025b](#bib.bib1 "CollabLLM: From passive responders to active collaborators"); Lu et al., [2025a](#bib.bib5 "DAMA: Data- and model-aware alignment of multi-modal LLMs"); Kwon et al., [2020](#bib.bib145 "Inverse rational control with partially observable continuous nonlinear dynamics")). Such settings inherently require multi-turn, long-horizon decision-making (Kim et al., [2024](#bib.bib15 "QuBE: Question-based belief enhancement for agentic LLM reasoning"); Wu et al., [2025a](#bib.bib14 "Agentic reasoning: A streamlined framework for enhancing LLM reasoning with agentic tools"); Hou et al., [2025](#bib.bib2 "T1: Advancing language model reasoning through reinforcement learning and inference scaling"); Lee and Kwon, [2024](#bib.bib133 "Episodic future thinking mechanism for multi-agent reinforcement learning")). However, long-horizon interaction remains brittle (Zhang et al., [2025b](#bib.bib53 "ReCAP: Recursive context-aware reasoning and planning for large language model agents"); Guan et al., [2025](#bib.bib44 "Evaluating LLM-based agents for multi-turn conversations: A survey")). Agents often exhibit _objective drift_—gradually deviating from the intended goal as interaction unfolds—and become token-inefficient by relying on ever-growing interaction histories to maintain intent (Huang et al., [2025](#bib.bib13 "Path drift in large reasoning models: How first-person commitments override safety"); Dongre et al., [2025](#bib.bib43 "Drift no more? Context equilibria in multi-turn LLM interactions")). In practice, objective drift is not merely a planning failure, but an interaction-level phenomenon where small execution errors accumulate over time, progressively misaligning the agent’s state with the original intent (Ichter et al., [2023](#bib.bib77 "Do as I can, not as I say: Grounding language in robotic affordances"); Yao et al., [2023](#bib.bib52 "ReAct: Synergizing reasoning and acting in language models")). This raises the question:

How can we build interactive LLM-based agents that reliably and efficiently solve long-horizon tasks?

Figure 1: Challenges in long-horizon interaction on the ScienceWorld using Llama-3.1 8B backbone. We compare ReAct (Yao et al., [2023](#bib.bib52 "ReAct: Synergizing reasoning and acting in language models")) (prompt-based, non-hierarchical), Glider (Hu et al., [2025](#bib.bib100 "Divide and conquer: Grounding LLMs as efficient decision-making agents via offline hierarchical reinforcement learning")) (hierarchical fine-tuning baseline), and our method (Multi2). (b) Token Efficiency: normalized inference-time token efficiency (higher is better), computed as performance/tokens and normalized such that the proposed method equals 1.0.

Recent studies mitigate such drift by decomposing long tasks into sub-goals via hierarchical planning (Hu et al., [2025](#bib.bib100 "Divide and conquer: Grounding LLMs as efficient decision-making agents via offline hierarchical reinforcement learning"); Prasad et al., [2024](#bib.bib57 "ADaPT: As-needed decomposition and planning with language models")). Yet, long-horizon interaction is not fully solved: decomposition can reduce planning failures, but execution can still break down over extended rollouts due to compounding errors and constraint-heavy actions (Erdogan et al., [2025](#bib.bib74 "Plan-and-Act: Improving planning of agents for long-horizon tasks"); Sinha et al., [2026](#bib.bib75 "The illusion of diminishing returns: Measuring long horizon execution in LLMs")). Moreover, many hierarchical agents implement role specialization primarily through prompting, which can blur as context accumulates, leaving agents vulnerable to constraint violations and loops (Li et al., [2025a](#bib.bib76 "Agent-oriented planning in multi-agent systems")). These observations reveal a more fundamental limitation: while hierarchical decomposition improves planning, existing approaches fail to ensure stable execution, as the executor is not explicitly trained to correct compounding errors during interaction (Dang et al., [2025](#bib.bib18 "Multi-agent collaboration via evolving orchestration"); Nisa et al., [2025](#bib.bib17 "Agentic AI: The age of reasoning—A review")). Accordingly, we argue that reliable multi-turn interaction requires (1) explicit task-level decomposition to preserve intent, (2) robust action execution that improves through interaction with the environment, and (3) token-efficient interaction.

To quantify these challenges, Figure [1](#S1.F1 "Figure 1 ‣ 1 Introduction ‣ Multi2: Hierarchical Multi-Agent Decision-Making with LLM-Based Agents in Interactive Environments") compares ReAct (Yao et al., [2023](#bib.bib52 "ReAct: Synergizing reasoning and acting in language models")), Glider (Hu et al., [2025](#bib.bib100 "Divide and conquer: Grounding LLMs as efficient decision-making agents via offline hierarchical reinforcement learning")), and Multi2 in ScienceWorld (Wang et al., [2022](#bib.bib84 "ScienceWorld: Is your agent smarter than a 5th grader?")) in terms of (a) horizon robustness and (b) token efficiency. Glider improves over ReAct via hierarchical decomposition, but its performance still degrades as horizons lengthen, indicating that long-horizon brittleness and higher token costs remain under multi-turn interaction. In contrast, Multi2 sustains higher performance at longer horizons while achieving better token efficiency, underscoring the need for reliable and efficient interactive LLM-based agents.

We therefore propose Multi2, a hierarchical multi-agent framework that decouples task decomposition (System 1) and action execution (System 2) through role specialization. Specifically, System 1 generates context-aware sub-goals that preserve global intent, while System 2 executes atomic actions in the dynamic environment, enabling selective invocation for token-efficient interaction. We train the two systems with a role-aligned pipeline: System 1 via SFT for structured sub-goal planning, and System 2 via offline-to-online RL to initialize stably from offline data and continually self-improve through online interaction.

In summary, our contributions are as follows: (1) We propose Multi2, a role-specialized hierarchical training-and-adaptation framework that decouples sub-goal planning and action execution as different optimization problems for interactive environments. This design improves goal consistency and mitigates objective drift in long-horizon interaction. (2) We introduce a principled offline-to-online RL objective for System 2, combining policy-anchored offline learning with Kullback-Leibler (KL)-regularized online refinement for stable self-improvement with a novel loss design. (3) We release well-structured hierarchical datasets for training and evaluating interactive LLM-based agents across multiple environments (see Section [5.1](#S5.SS1 "5.1 Experiment Setups ‣ 5 Experiments ‣ Multi2: Hierarchical Multi-Agent Decision-Making with LLM-Based Agents in Interactive Environments")). (4) Multi2 consistently outperforms strong baselines in performance, token efficiency, and long-horizon robustness.

## 2 Related Works

#### Hierarchical Structure for Interactive LLM-Based Agents.

There has been growing interest in applying AI systems to interactive environments (Kim et al., [2025b](#bib.bib124 "OASIS: Open-world adaptive self-supervised and imbalanced-aware system"), [a](#bib.bib129 "Improving network attack classification on imbalanced real-world intrusion incident datasets"); Joe et al., [2026](#bib.bib122 "Contrastive learning based network attack classifier for imbalanced data"); Kim et al., [2025c](#bib.bib123 "Personalized split federated learning with early exit: Pretraining and online learning against label shifts")), where agents must perceive changing states, make sequential decisions, and adapt their behavior over time (Park et al., [2026](#bib.bib119 "Fed-ADE: Adaptive learning rate for federated post-adaptation under distribution shift"); Kim et al., [2026a](#bib.bib120 "Evolving intelligent network attack classifier under label distribution shift"); Park et al., [2025b](#bib.bib131 "Personalized federated sensing for heterogeneous environment")). In this context, recent works have explored LLM-based agents that interact with external environments (e.g., embodied simulators and text games) (Dong et al., [2025](#bib.bib3 "Enhancing decision-making of large language models via actor-critic"); Wei et al., [2024](#bib.bib82 "Editable scene simulation for autonomous driving via collaborative LLM-agents"); Huang et al., [2024](#bib.bib83 "DriVLMe: enhancing LLM-based autonomous driving agents with embodied and social experiences"); Kim et al., [2026b](#bib.bib148 "SToRM: Supervised token reduction for multi-modal LLMs toward efficient end-to-end autonomous driving")). While such settings demand long-horizon, multi-turn interaction, many existing systems primarily target short-horizon tasks, leaving robust decision-making as an open challenge (Huang et al., [2025](#bib.bib13 "Path drift in large reasoning models: How first-person commitments override safety"); Dongre et al., [2025](#bib.bib43 "Drift no more? Context equilibria in multi-turn LLM interactions"); Park et al., [2025a](#bib.bib125 "ASAP: Unsupervised post-training with label distribution shift adaptive learning rate")). Common approaches build on sequential decision-making paradigms, such as ReAct (Yao et al., [2023](#bib.bib52 "ReAct: Synergizing reasoning and acting in language models")) and Reflexion (Shinn et al., [2023](#bib.bib54 "Reflexion: Language agents with verbal reinforcement learning")), which often rely on long interaction histories and can be token-inefficient. Moreover, these approaches are susceptible to objective drift when the initial intent or plan falls outside the context window (An et al., [2024](#bib.bib49 "Make your LLM fully utilize the context"); Tian et al., [2024](#bib.bib45 "Toward self-improvement of LLMs via imagination, searching, and criticizing"); Yang et al., [2025b](#bib.bib46 "The lighthouse of language: Enhancing LLM agents via critique-guided improvement"); Ngo et al., [2024](#bib.bib47 "The alignment problem from a deep learning perspective")).

To address these challenges, recent studies introduce hierarchical structures that decouple high-level task decomposition from low-level execution (Prasad et al., [2024](#bib.bib57 "ADaPT: As-needed decomposition and planning with language models"); Hu et al., [2025](#bib.bib100 "Divide and conquer: Grounding LLMs as efficient decision-making agents via offline hierarchical reinforcement learning"); Zhou et al., [2024](#bib.bib98 "ArCHer: Training language model agents via hierarchical multi-turn RL"); Rheey and Park, [2025](#bib.bib146 "Robust hierarchical anomaly detection using feature impact in iot networks"); Shek and Tokekar, [2025](#bib.bib101 "Option discovery using LLM-guided semantic hierarchical reinforcement learning"); Yang et al., [2025c](#bib.bib102 "Leveraging imitation learning and LLMs for efficient hierarchical reinforcement learning")). For instance, ADaPT (Prasad et al., [2024](#bib.bib57 "ADaPT: As-needed decomposition and planning with language models")) proposes a hierarchical approach that explicitly plans and decomposes complex sub-tasks. Glider (Hu et al., [2025](#bib.bib100 "Divide and conquer: Grounding LLMs as efficient decision-making agents via offline hierarchical reinforcement learning")) introduces a hierarchical RL framework that leverages intermediate sub-goals to ease credit assignment in long-horizon settings. However, these approaches primarily focus on decomposition, leaving execution robustness largely underexplored. In particular, the executor is typically not optimized to correct compounding errors during interaction, which limits their effectiveness in long-horizon settings (Erdogan et al., [2025](#bib.bib74 "Plan-and-Act: Improving planning of agents for long-horizon tasks"); Sinha et al., [2026](#bib.bib75 "The illusion of diminishing returns: Measuring long horizon execution in LLMs")). Our work addresses this gap by explicitly coupling hierarchical decomposition with a learning-based execution policy that improves through interaction.

#### Offline-to-Online Reinforcement Learning.

A standard approach for adapting LLM-based agents is SFT (Li et al., [2024](#bib.bib60 "Getting more juice out of the SFT data: Reward learning from human demonstration improves SFT for LLM alignment")), but it can overfit to domain-specific trajectories and generalize poorly beyond the training data (Szot et al., [2024](#bib.bib73 "Large language models as generalizable policies for embodied tasks"); Ji et al., [2025](#bib.bib65 "Unlocking LLMs’ self-improvement capacity with autonomous learning for domain adaptation"); Li et al., [2025b](#bib.bib63 "Data mixing optimization for supervised fine-tuning of large language models"); Lu et al., [2025b](#bib.bib64 "Fine-tuning large language models for domain adaptation: Exploration of training strategies, scaling, model merging and synergistic capabilities")). RL provides an alternative by improving behavior through interaction rather than blind imitation (Ma et al., [2024](#bib.bib69 "Coevolving with the other you: Fine-tuning LLM with sequential cooperative multi-agent reinforcement learning"); Lee and Kwon, [2025b](#bib.bib132 "Instant inverse modeling of stochastic driving behavior with deep reinforcement learning"); Jia et al., [2025](#bib.bib68 "Controlling large language model with latent action"); Lee and Kwon, [2023b](#bib.bib139 "Stability analysis in mixed-autonomous traffic with deep reinforcement learning"); Lin et al., [2025](#bib.bib70 "QLASS: Boosting language agent inference via Q-guided stepwise search"); Lee and Kwon, [2025d](#bib.bib127 "Temporal distance-aware transition augmentation for offline model-based reinforcement learning")). However, purely online RL tends to be sample-inefficient and brittle, and can become costly due to unsafe exploration during rollouts (Wei et al., [2017](#bib.bib38 "Online reinforcement learning in stochastic games"); Ball et al., [2023](#bib.bib40 "Efficient online reinforcement learning with offline data"); Wagenmaker and Pacchiano, [2023](#bib.bib41 "Leveraging offline data in online reinforcement learning"); Lillicrap et al., [2016](#bib.bib37 "Continuous control with deep reinforcement learning"); Lee and Kwon, [2023a](#bib.bib136 "Foresighted decisions for inter-vehicle interactions: an offline reinforcement learning approach"); Fujimoto et al., [2018](#bib.bib39 "Addressing function approximation error in actor-critic methods")), whereas purely offline RL is constrained by dataset quality and distributional coverage (Eom et al., [2024](#bib.bib134 "Selective imitation for efficient online reinforcement learning with pre-collected data"); Kostrikov et al., [2022](#bib.bib174 "Offline reinforcement learning with implicit Q-learning"); Eom and Kwon, [2026](#bib.bib121 "Price of the autonomous strategy with reinforcement learning in mixed-autonomy traffic networks"); Eo et al., [2024](#bib.bib137 "The impact of dataset on offline reinforcement learning performance in uav-based emergency network recovery tasks"); Kim et al., [2022](#bib.bib144 "Curriculum reinforcement learning for cohesive team in mobile ad hoc networks")), limiting improvements beyond the behavior policy (Lee et al., [2024](#bib.bib135 "AD4RL: Autonomous driving benchmarks for offline reinforcement learning with value-based dataset"); Lee and Kwon, [2025a](#bib.bib130 "Episodic future thinking with offline reinforcement learning for autonomous driving"); Wang et al., [2024](#bib.bib165 "A unified principle of pessimism for offline reinforcement learning under model mismatch"); Park et al., [2024](#bib.bib166 "Is value learning really the main bottleneck in offline RL?")).

These trade-offs are exacerbated for long-horizon LLM-based agents, where dynamic states and constraint-heavy actions are difficult to cover with static datasets. Recent work has explored offline-to-online RL, using offline data to stably initialize a policy without online interaction, then improving it through online interaction (Zhang et al., [2025a](#bib.bib10 "Beyond online sampling: Bridging offline-to-online alignment via dynamic data transformation for LLMs"); Bai et al., [2024](#bib.bib99 "DigiRL: Training in-the-wild device-control agents with autonomous reinforcement learning"); Ivison et al., [2024](#bib.bib29 "Unpacking DPO and PPO: Disentangling best practices for learning from preference feedback"); Lanchantin et al., [2025](#bib.bib30 "Bridging offline and online reinforcement learning for LLMs"); Lee and Kwon, [2025c](#bib.bib126 "Scenario-free autonomous driving with multi-task offline-to-online reinforcement learning"); Park et al., [2025c](#bib.bib147 "Test-time fine-tuning of image compression models for multi-task adaptability")). While prior offline-to-online RL methods focus on stabilizing policy improvement, they are not tailored to the structured execution requirements induced by hierarchical agents. In contrast, we design an objective that explicitly aligns with the role of the executor, enabling stable and sample-efficient learning, while supporting continual self-improvement.

## 3 Problem Setups

### 3.1 Problem Setup for LLM-Based Agents

We denote the high-level agent as System 1 for context-aware planning and the low-level agent as System 2 for action execution. Each agent is modeled as a partially observable Markov decision process tailored to its role. At each high-level step, System 1 takes a task description Kn∈N and the observation 𝐨t as input and outputs a sub-goal gh. Conditioned on 𝐨t and gh, System 2 executes an atomic action 𝐚t to accomplish the sub-goal gh.

Figure 2: Overview of Multi2. (a) Offline Training: System 1 (high-level agent) is trained via SFT on the high-level dataset 𝒟sys1 to generate sub-goals, while System 2 (low-level agent) is initialized via offline RL on the low-level dataset 𝒟sys2 using an actor-critic objective. (b) Online Training: starting from the offline policy, System 2 continues self-improvement through interaction, updating its policy while retaining and augmenting the replay buffer. (c) Execution: at inference time, System 1 proposes context-aware sub-goals and System 2 executes atomic actions conditioned on each sub-goal; System 1 is re-invoked when a sub-goal is achieved.

### 3.2 Sequential Decision-Making Problem

We formulate the hierarchical LLM-based agents as a partially observable Markov decision process. It is defined by the tuple ⟨𝒮,𝒜,𝒪,𝒯,Ω,ℛ,γ⟩, where 𝐬t∈𝒮 is a state, 𝐚t∈𝒜 is an action, 𝐨t∈𝒪 is an observation. The state transition probability is given by 𝒯:𝒮×𝒜→𝒮, the observation probability by Ω:𝒮→𝒪, the reward function by ℛ:𝒮×𝒜×𝒮→ℝ, and γ∈\[0,1) denotes a temporal discounted factor.

The objective of RL is to learn a policy that maximizes the expected cumulative return 𝒥π\=𝔼π​\[∑tγt​rt\], where rt\=ℛ​(𝐬t,𝐚t,𝐬t+1). We adopt an actor-critic structure, where the actor selects an action 𝐚t according to the policy πθ​(𝐚t∣𝐨t), while the critic estimates the action-value Qω​(𝐨t,𝐚t)\=𝔼π​\[∑fγf​rt+f∣𝐨t,𝐚t\] and the state-value Vψ​(𝐨t)\=𝔼πθ​\[Qω​(𝐨t,𝐚t)\]. Accordingly, the advantage function is defined as A​(𝐨t,𝐚t)\=Qω​(𝐨t,𝐚t)−Vψ​(𝐨t), which measures how much better a specific action is than the expected policy value at 𝐨t. Within the actor-critic framework, the actor updates its policy using the value estimates from the critic, while the critic learns by minimizing the temporal-difference (TD) error.

## 4 The Proposed Multi2 Framework

We propose Multi2, a hierarchical multi-agent decision-making framework for long-horizon interactive tasks (Figure [2](#S3.F2 "Figure 2 ‣ 3.1 Problem Setup for LLM-Based Agents ‣ 3 Problem Setups ‣ Multi2: Hierarchical Multi-Agent Decision-Making with LLM-Based Agents in Interactive Environments")). Multi2 couples hierarchical control with an offline-to-online RL training pipeline. Given a task Kn, System 1 generates a sub-goal gh∼πϕ(⋅∣𝐨t;Kn), which specifies a sub-task for System 2. Conditioned on the current observation and sub-goal, System 2 selects an atomic action 𝐚t∼πθ(⋅∣𝐨t;gh) to accomplish gh.111Here, ϕ and θ denote the parameters of System 1 and System 2, respectively. Once gh is achieved (or terminated), System 2 queries System 1 to generate the next sub-goal gh+1, and the process repeats.

### 4.2 Role-Specialized Offline Training for System 1 and System 2

To train two systems efficiently, we adopt role-specific objectives aligned with their roles. System 1 focuses on semantic, context-aware decision-making, while System 2 is responsible for atomic action selection. The overall procedure, including the training of System 1 and System 2, is summarized in Algorithm [1](#alg1 "Algorithm 1 ‣ 4.3 Online Training for System 2 Self-Improvement ‣ 4 The Proposed Multi2 Framework ‣ Multi2: Hierarchical Multi-Agent Decision-Making with LLM-Based Agents in Interactive Environments").

System 1 serves as a high-level planner that generates sub-goals for each task Kn. Since sub-goal generation primarily requires consistent semantic planning, we train System 1 with SFT to provide a stable supervision signal and encourage task-level consistency. Specifically, we fine-tune the policy πϕ by behavior cloning on the high-level dataset 𝒟sys1.

The weight wt is computed from the advantage as wt\=exp⁡(1α​A​(𝐨t,𝐚t;gh)), so that actions with higher estimated advantage are reinforced more strongly. Our key design is the KL regularization term, which constrains πθon to stay close to the offline-trained policy πθoff. This regularization prevents overly sharp policy shifts during online refinement, mitigates mode collapse (Chen et al., [2026](#bib.bib21 "KL-regularized reinforcement learning is designed to mode collapse"); Brown et al., [2025](#bib.bib22 "KL-regularised Q-learning: A token-level action-value perspective on online RLHF"); Li et al., [2026](#bib.bib23 "The choice of divergence: A neglected key to mitigating diversity collapse in reinforcement learning with verifiable reward")), and stabilizes self-improvement. The coefficient η trades off policy improvement against this stability constraint.

## 5 Experiments

We consider how to build interactive LLM-based agents that solve long-horizon tasks reliably and efficiently. Such settings are challenging because errors compound over multi-turn interaction, often causing objective drift and increased token usage. To evaluate whether Multi2 addresses these issues, we benchmark it in interactive environments that require agentic capabilities—i.e., maintaining task intent, decomposing goals, and executing valid actions under changing observations. Additional results are provided in Appendix [G](#A7 "Appendix G Additional Results and Analyses ‣ Multi2: Hierarchical Multi-Agent Decision-Making with LLM-Based Agents in Interactive Environments").

Accordingly, our experiments aim to answer three questions: (1) Does Multi2 improve long-horizon task performance across diverse interactive benchmarks? (2) How token-efficient is Multi2 at inference time compared to prior works? (3) Which components (role specialization, loss design, and offline-to-online training) are responsible for the gains?

### 5.1 Experiment Setups

#### Benchmarks.

We evaluate our framework on ScienceWorld (Wang et al., [2022](#bib.bib84 "ScienceWorld: Is your agent smarter than a 5th grader?")), ALFWorld (Shridhar et al., [2021](#bib.bib85 "ALFWorld: Aligning text and embodied environments for interactive learning")), and TextCraft (Prasad et al., [2024](#bib.bib57 "ADaPT: As-needed decomposition and planning with language models")), which cover complementary observation/action spaces for multi-turn interaction. ScienceWorld tests curriculum-driven, long-horizon planning in a stateful environment where actions update the world, and agents adapt to new observations. ALFWorld targets household manipulation with discrete high-level actions and sparse rewards, stressing goal maintenance and credit assignment. TextCraft evaluates recipe-based symbolic crafting, where sparse terminal rewards and compositional dependencies challenge agents to sustain intent and avoid objective drift. Additional details are in Appendix [B](#A2 "Appendix B Benchmarks ‣ Multi2: Hierarchical Multi-Agent Decision-Making with LLM-Based Agents in Interactive Environments").

#### Backbones.

Our framework is built on the instruction-tuned open-source Qwen-2.5 3B (Yang et al., [2025a](#bib.bib27 "Qwen2.5 technical report")), Mistral 7B v0.3 (Jiang et al., [2023](#bib.bib26 "Mistral 7b")), and Llama-3.1 8B (Meta AI, [2024](#bib.bib25 "Introducing meta llama 3: the most capable openly available llm to date")) models. Training is conducted on an RTX PRO 6000 Blackwell (128GB RAM), and details of the experimental setup and hyperparameters are provided in Appendix [C](#A3 "Appendix C Model Structure ‣ Multi2: Hierarchical Multi-Agent Decision-Making with LLM-Based Agents in Interactive Environments").

#### Model Architecture with Low-Rank Adaptation.

We adopt low-rank adaptation (LoRA) (Hu et al., [2022](#bib.bib92 "LoRA: Low-rank adaptation of large language models")) for parameter-efficient fine-tuning, enabling role specialization without updating the full backbone. System 1 and System 2 share the same pretrained backbone but use separate LoRA adapters. During both training and inference, only the adapter corresponding to the selected system is activated.

#### Datasets.

Since training System 1 and System 2 requires hierarchical datasets, we build our dataset construction pipeline on the framework introduced in Glider (Hu et al., [2025](#bib.bib100 "Divide and conquer: Grounding LLMs as efficient decision-making agents via offline hierarchical reinforcement learning")) and further modify and extend it to improve consistency and reproducibility. In particular, we refine sub-goal extraction and prompt construction with code-based rules, improving reproducibility and reducing sensitivity to design and prompting choices. As a result, the datasets are more consistent and readily reusable for training and evaluating long-horizon hierarchical agents. We release the datasets and code to support broader adoption and fair comparison. Further details are provided in Appendix [A](#A1 "Appendix A Dataset Construction ‣ Multi2: Hierarchical Multi-Agent Decision-Making with LLM-Based Agents in Interactive Environments").

#### Metric.

We adopt a strict pass@1 metric, allowing only a single rollout without retries or resampling, to emphasize reliability and one-shot efficiency. We report pass@1 separately on ID and OOD splits defined at the task-structure level (Appendix [D](#A4 "Appendix D Splits and Evaluation Protocol ‣ Multi2: Hierarchical Multi-Agent Decision-Making with LLM-Based Agents in Interactive Environments")). ID tasks share goal templates and interaction patterns with those seen during training, whereas OOD tasks use novel templates. Importantly, evaluation is conducted in newly instantiated episodes with different object instances, layouts, and intermediate states, so even ID tasks are not identical to training trajectories and require state-level generalization. For prompt-based baselines, we follow standard protocols with a small set of in-context demonstrations. For fine-tuned agents, we evaluate zero-shot without in-context examples.

Reflexion (Shinn et al., [2023](#bib.bib54 "Reflexion: Language agents with verbal reinforcement learning")), however, is a multi-trial method that improves through self-critique and revision across successive attempts. Accordingly, we report pass@6, consistent with its standard evaluation protocol.

### 5.3 Performance Comparison

Table [1](#S5.T1 "Table 1 ‣ Datasets. ‣ 5.1 Experiment Setups ‣ 5 Experiments ‣ Multi2: Hierarchical Multi-Agent Decision-Making with LLM-Based Agents in Interactive Environments") summarizes overall results on interactive environments. Notably, Multi2 achieves the strongest performance across most backbones and splits, suggesting improved generalization across model settings. Prompt-based baselines (e.g., ReAct and Reflexion) perform poorly, which is consistent with compounding errors and drift in long-horizon tasks. Here, ADaPT outperforms GRPO on most tasks because its hierarchical prompting provides a useful direction. Among fine-tuning-based baselines, Glider improves over prompt-based methods, indicating that hierarchical structure and parameter updates help mitigate long-horizon brittleness. However, Glider relies primarily on prompt-based role separation while sharing a single adapter across the planner and executor, which can limit role specialization compared to a fully decoupled design. In contrast, Multi2 addresses this challenge by decoupling System 1 and System 2 with role-specialized training, and by optimizing System 2 via an RL pipeline with the proposed training pipeline and well-structured loss function.

In contrast, prompt-based agents often persist in rejected commands or repeat actions (Figures [3](#S5.F3 "Figure 3 ‣ Metric. ‣ 5.1 Experiment Setups ‣ 5 Experiments ‣ Multi2: Hierarchical Multi-Agent Decision-Making with LLM-Based Agents in Interactive Environments")(a–c)), indicating limited recovery once trajectories deviate. Fine-tuning-based agents likewise tend to commit to an implausible target and continue with non-progressing or unsupported actions (Figures [3](#S5.F3 "Figure 3 ‣ Metric. ‣ 5.1 Experiment Setups ‣ 5 Experiments ‣ Multi2: Hierarchical Multi-Agent Decision-Making with LLM-Based Agents in Interactive Environments")(d–e)). ## 6 Conclusion

We propose Multi2, a hierarchical multi-agent framework for long-horizon interactive environments, explicitly addressing two practical bottlenecks of LLM-based agents: objective drift and poor token efficiency. Our key contribution is a principled decoupling of planning and execution, where System 1 generates context-aware sub-goals while System 2 executes atomic actions conditioned on them, enabling stable coordination over long horizons. To support this structure, we introduce a unified training pipeline that combines policy-anchored offline RL with KL-regularized online refinement, providing a stable mechanism for continual improvement under multi-turn interaction. Across diverse interactive benchmarks, Multi2 consistently improves performance while yielding better token efficiency and stronger robustness to objective drift. We also release our processed hierarchical datasets, enabling broader adoption in the community.

## Impact Statement

This paper advances methods for agentic AI–LLM-based agents that can reliably and efficiently interact with dynamic environments over long-horizons. By contributing to more effective and principled learning algorithms, this work has the potential to benefit a wide range of applications and, when deployed responsibly, improve productivity and quality of life. As with most general-purpose machine learning techniques, the proposed methods could be applied in both beneficial and potentially harmful contexts depending on the application domain and deployment practices. We do not identify ethical concerns or negative societal consequences unique to this work beyond those commonly associated with broader deployment of capable interactive AI systems. Responsible use of the proposed methods should follow established best practices regarding safety, fairness, transparency, and accountability.

## Appendix A Dataset Construction

We describe the dataset construction pipeline for the role-specific hierarchical datasets. Building on Glider \[[17](#bib.bib100 "Divide and conquer: Grounding LLMs as efficient decision-making agents via offline hierarchical reinforcement learning")\], we extend the processing to better match our hierarchical setting. Concretely, we generate the dataset programmatically via a code-based pipeline that loads GPT-4-Turbo conversation trajectories released by AgentGym \[[77](#bib.bib87 "AgentGym: Evolving large language model-based agents across diverse environments")\] and converts them into role-specific training instances. We parse into supervision signals aligned with high-level planning and low-level execution.

We make two modifications to improve stability and reproducibility: deterministic, rule-based sub-goal extraction (e.g., step segmentation, command normalization, and filtering non-actionable text) and standardized prompt templates per role with fixed instruction headers and output schemas to reduce prompt sensitivity. Finally, we convert each episode into (1) observation-sub-goal pairs for System 1 and (2) sub-goal-conditioned low-level transitions for System 2.

### A.2 Role-specific Prompts

We use separate prompts to reflect the distinct input-output requirements of the planner (System 1) and executor (System 2): System 1 maps {task, observation} to a sub-goal, while System 2 maps {sub-goal, observation} to an executable action and a completion signal.

System 1 Prompt. You are a high-level planner. Based on the state (task description and current observation), please generate a clear and simple sub-goal. `\n` Task description: `\n` Grouped action: \[\]. Current observation: `\n`

System 2 Prompt. You are a low-level action executor. Based on the current sub-goal and observation, please generate an executable action and determine whether the sub-goal has been completed (true/false). `\n` Sub-goal: `\n` Observation: `\n`

### A.3 Reward Shaping

In ScienceWorld \[[71](#bib.bib84 "ScienceWorld: Is your agent smarter than a 5th grader?")\], the environment provides a dense progress reward in \[0,1\] (along with a success flag) reflecting partial completion of the underlying sub-goals; we follow the official reward definition and directly use the environment-provided reward signal.

In ALFWorld \[[66](#bib.bib85 "ALFWorld: Aligning text and embodied environments for interactive learning")\], the environment reward is sparse: the agent receives a task-level reward of 1 only upon completing the full task, and 0 otherwise. To enable hierarchical training, we introduce an internal sub-goal-level shaping reward used only for training System 2. Given the extracted sub-goal sequence, we segment each trajectory into contiguous intervals where one sub-goal is active, and assign an internal reward of 1 when the active sub-goal is completed (as detected by our trajectory parser), and 0 otherwise. Importantly, the environment reward remains unchanged: the task-level reward is provided only when the full task succeeds.

In TextCraft \[[61](#bib.bib57 "ADaPT: As-needed decomposition and planning with language models")\], the environment is also sparse: the agent receives a reward of 1 only upon successfully crafting the target item (and 0 otherwise), under a constrained command interface (e.g., craft/get/inventory). We therefore apply the same hierarchical shaping strategy, awarding intermediate reward when the currently active sub-goal (e.g., the next recipe step/prerequisite) is satisfied, while preserving the original terminal reward for task completion.

### A.4 Reproducibility and Release

All sub-goal extraction rules, prompt templates, and preprocessing scripts are implemented as deterministic, code-based transformations, minimizing sensitivity to manual design choices and ensuring that the resulting datasets are consistent and reusable. We publicly release the processed role-specific datasets, the exact prompt templates and extraction rules, as well as the training and evaluation code at [https://anonymous-projectpage.github.io/Multi-Square/](https://anonymous-projectpage.github.io/Multi-Square/). This release enables reproducible training and transparent evaluation, supporting fair comparison across long-horizon LLM agent methods.

## Appendix B Benchmarks

We evaluate long-horizon decision-making on three benchmarks that differ in dynamics, action constraints, and feedback signals. ScienceWorld \[[71](#bib.bib84 "ScienceWorld: Is your agent smarter than a 5th grader?")\] provides a score considering incremental progress, whereas ALFWorld \[[66](#bib.bib85 "ALFWorld: Aligning text and embodied environments for interactive learning")\] and TextCraft \[[61](#bib.bib57 "ADaPT: As-needed decomposition and planning with language models")\] are primarily evaluated by success rate.

### B.1 ScienceWorld

Overview. ScienceWorld \[[71](#bib.bib84 "ScienceWorld: Is your agent smarter than a 5th grader?")\] is a text-only interactive world aimed at measuring scientific reasoning in the context of simple, experiment-like procedures. The benchmark consists of 30 tasks drawn from 10 topical areas and includes comparatively rich simulated phenomena (e.g., circuits, phase changes, chemistry, and biology), making it more stateful than many standard text-game settings.

Task Example. Solving a task usually requires carrying out multiple steps with appropriate tools—for instance, assembling a circuit to check conductivity, inducing and verifying a phase transition (melting/boiling), or following a sequence of lab-style operations.

ScienceWorld Task Example. Task: Turn on the green light bulb. First, focus on the green light bulb. Then, create an electrical circuit that powers it on.

Observation. At each step, the agent receives a natural-language description that includes the current location, visible objects (often with attributes), and outcome feedback from the previous action (e.g., inventory changes or success/failure messages). Because relevant information only becomes available after navigation and interaction, the environment is effectively partially observable.

Action. The agent issues executable textual commands grounded in entities (e.g., focus on, pick up, open/close, connect, move ... to ...). ScienceWorld exposes 25 abstract action templates that are instantiated with object arguments, and it supports a valid-action checking mechanism that can enumerate which commands are admissible under the current state constraints.

Reward. Rather than a purely terminal reward, ScienceWorld returns a scalar SCORE that increases as the agent makes progress and reaches 1.0 upon completing the task. We report ScienceWorld results using this score as the performance metric.

### B.2 ALFWorld

Overview. ALFWorld \[[66](#bib.bib85 "ALFWorld: Aligning text and embodied environments for interactive learning")\] is a text-only interactive benchmark derived from the embodied ALFRED household domain, implemented on top of TextWorld. It targets long-horizon household tasks that require both navigation and object manipulation under textual observations and action commands.

Task Example. The benchmark spans six high-level task categories, including pick-and-place objectives and multi-step routines such as cleaning, heating, or cooling an object before placing it at a target location (e.g., “place a clean mug on a desk”).

ALFWorld Task Example. Task: Clean some spatula and put it in diningtable.

Observation. At each step, the agent receives a natural-language description of the current room and the set of visible objects and receptacles, together with the goal instruction. Typically, the agent begins with a list of room contents and must then navigate across locations, open containers, and manipulate objects to satisfy the goal.

Action. The action space consists of template-based text commands covering navigation and manipulation, such as go to <receptacle> and take <obj> from <receptacle>, as well as task-specific interactions (e.g., use <tool>). For the text-game setting, the environment can also provide the set of currently admissible commands (e.g., info\[’admissible\_commands’\]), making action validity constraints explicit.

Reward. ALFWorld provides sparse feedback, where success is determined by completing the full instruction. We report results using task success (success rate), following standard evaluation practice.

### B.3 TextCraft

Overview. TextCraft \[[61](#bib.bib57 "ADaPT: As-needed decomposition and planning with language models")\] is a text-based crafting benchmark proposed by ADaPT \[[61](#bib.bib57 "ADaPT: As-needed decomposition and planning with language models")\], motivated by Minecraft-style recipe systems. Compared to navigation-centric environments, TextCraft primarily stresses compositional goal decomposition: achieving the final target typically requires crafting a sequence of prerequisite items recursively.

Task Example. A typical instance asks the agent to produce a high-level item (e.g., beehive) that depends on intermediate ingredients (e.g., honeycomb and planks), which may themselves require multiple lower-level crafting steps.

TextCraft Task Example. Task: Craft polished blackstone wall. Crafting commands: • craft 6 blackstone slab using 3 blackstone • craft 4 polished blackstone stairs using 6 polished blackstone • craft 1 polished blackstone pressure plate using 2 polished blackstone • craft 6 blackstone wall using 6 blackstone • craft 4 polished blackstone using 4 blackstone • craft 6 polished blackstone wall using 6 polished blackstone • craft 1 polished blackstone button using 1 polished blackstone • craft 4 polished blackstone bricks using 4 polished blackstone • craft 4 blackstone stairs using 6 blackstone • craft 6 polished blackstone slab using 3 polished blackstone

Observation. The environment provides the target item together with the current crafting context, including recipe information and textual feedback from executed actions. Since state transitions are deterministic, the agent can track progress via explicit messages (e.g., Got ... or Crafted ...) and by maintaining an internal inventory state.

Action. The interaction space consists of commands for gathering resources and applying recipes, such as get <item> and structured crafting instructions (e.g., craft 4 oak planks using 1 oak log). In some cases, the environment permits category-level references (e.g., requesting “planks”), requiring the agent to resolve the request into a valid concrete item (e.g., oak planks).

Reward. TextCraft is scored by whether the agent completes the target objective, and we report the success rate accordingly. The benchmark includes tasks with recipe dependency depth ranging from 2 to 4, and the official evaluation includes a 200-task test suite spanning these depths.

### B.4 Qualitative Comparison

The three benchmarks probe complementary abilities required for long-horizon agents.

*   •
    
    ScienceWorld \[[71](#bib.bib84 "ScienceWorld: Is your agent smarter than a 5th grader?")\]: This benchmark tests stateful interaction under rich dynamics, requiring agents to reason about changing object states (e.g., heating, mixing, connecting) and to issue valid, grounded commands from a large, constraint-driven action space, while receiving shaped score feedback.
    
*   •
    
    ALFWorld \[[66](#bib.bib85 "ALFWorld: Aligning text and embodied environments for interactive learning")\]: This benchmark emphasizes goal-directed household execution; agents must navigate across rooms and manipulate objects, yet evaluation is primarily by final task success, making recovery from small mistakes critical. Moreover, many objects have indexed identifiers, so selecting the wrong instance can cause failure even when the action is semantically correct.
    
*   •
    
    TextCraft \[[61](#bib.bib57 "ADaPT: As-needed decomposition and planning with language models")\]: This benchmark focuses on _compositional planning_; although interaction templates are relatively simple, agents must recursively decompose a target recipe into prerequisite sub-recipes (depth up to 4) and accurately track ingredients, stressing long-horizon planning and progress bookkeeping.
    

### B.5 Out-of-Scope: Static Math and Code Generation Benchmarks

Our study targets long-horizon decision-making, where an agent repeatedly interprets partial observations, selects an atomic action under environment constraints, and receives feedback through state transitions and rewards. This setting is central to the objective drift and robustness issues we analyze, as errors can compound over multi-turn interaction and invalid actions can stall progress. \[[71](#bib.bib84 "ScienceWorld: Is your agent smarter than a 5th grader?"), [66](#bib.bib85 "ALFWorld: Aligning text and embodied environments for interactive learning"), [61](#bib.bib57 "ADaPT: As-needed decomposition and planning with language models")\]

#### Why Math and Coding Benchmarks are Not a Good Fit.

While math and code-generation benchmarks are valuable for evaluating reasoning and synthesis, they are not well aligned with our goal of assessing _agentic_ behavior in interactive environments. Our setting centers on a closed-loop state-action process, where each action changes the environment and the agent must continually re-ground its decisions in new observations over multiple steps. In contrast, most math/coding benchmarks primarily evaluate producing a correct final output under a static specification, so they do not stress key failure modes in interactive agents such as constraint-violating actions, compounding execution errors, and recovery from partial progress. Accordingly, we focus on interactive benchmarks that explicitly test long-horizon decision-making under evolving world states.

#### Scope Statement.

Accordingly, we restrict evaluation to interactive environments that explicitly instantiate the observation-action-feedback loop and expose long-horizon failures such as compounding errors, invalid-action spirals, and objective drift.

## Appendix D Splits and Evaluation Protocol

### D.1 In-Distribution / Out-of-Distribution Splits

To evaluate generalization under long-horizon interaction, we report results on ID and OOD splits across benchmarks. Here, the distinction is defined at the task-structure level: an evaluation instance is considered ID split if it follows a goal template and interaction pattern seen during training (e.g., the same type of sub-goal decomposition and action schema used to build our role-specific datasets), and it is considered OOD split if it follows a held-out template.

#### Split Construction.

For each benchmark, we construct the ID and OOD splits by matching each evaluation task to the task specifications present in the training corpus. The ID split consists of tasks whose goal descriptions (or templates) are observed during training, whereas the OOD split consists of held-out tasks that are never used for training in any form. Importantly, even within the ID split, evaluation episodes are not fixed: the agent may start from different locations, and object instances may appear in different positions or with different identifiers due to environment stochasticity. As a result, success on the ID split still requires robust state grounding and long-horizon control, rather than reproducing memorized action sequences.

#### ScienceWorld Split Refinement.

ScienceWorld provides official split annotations for evaluation; however, for a controlled analysis consistent with the training-set exposure definition above, we additionally construct a strict ID and OOD partition by checking whether each task specification appears in our training corpus. This refinement is necessary because tasks with different split labels can share overlapping templates or instances, which can blur the boundary between ID and OOD when evaluation is performed solely using the provided annotations. We therefore use this refined partition for all ScienceWorld results reported in the paper to obtain a clearer assessment of generalization.

### D.2 Prompting Protocol and In-Context Learning

We distinguish between prompt-based baselines that rely on in-context learning (few-shot demonstrations) and fine-tuning-based agents trained by updating model parameters. Unless otherwise stated, we evaluate prompt-based methods with a small set of in-context demonstrations following their original protocols, which increases input length but helps induce the intended interaction format. For fine-tuned agents, including Multi2, we do not provide in-context demonstrations at inference time; in particular, Multi2 is evaluated in a zero-shot setting and relies on role-specialized training to directly produce sub-goals and executable actions.

### D.3 Task Difficulty

We define task difficulty in ScienceWorld based on the average interaction length required by our agent to solve each task. Intuitively, tasks that require more environment steps tend to involve long-horizon decision-making and provide more probabilities for compounding errors. Concretely, for each task, we run our agent across multiple evaluation episodes and compute the average number of environment steps taken until termination (success or failure). We then sort tasks by this average interaction length and partition them into three groups of similar size: easy (short), medium, and hard (long).

Figure 16: Task difficulty based on average interaction length in the ScienceWorld benchmark. For each task, we compute the average number of environment steps taken by our agent across evaluation episodes, and group tasks into Easy/Medium/Hard by quantiles of this length (short/medium/long horizon).

Figure [16](#A4.F16 "Figure 16 ‣ D.3 Task Difficulty ‣ Appendix D Splits and Evaluation Protocol ‣ Multi2: Hierarchical Multi-Agent Decision-Making with LLM-Based Agents in Interactive Environments") visualizes the distribution of average interaction lengths used to define task difficulty on (a) In-distribution Split and (b) Out-of-distribution Split. In both splits, easy tasks concentrate in shorter-length bins, medium tasks occupy intermediate ranges, and hard tasks are dominated by the longest interactions. Notably, the OOD split is shifted toward longer horizons overall, with a larger mass in the highest-length region (around 45–50 steps) compared to the ID split.

## Appendix E Baselines

All baselines use the same backbone LLMs as Multi2 and interact with the identical environment interface (observation formatting, action space, termination conditions, and maximum interaction budget). Across all baselines, we keep (1) the backbone model, (2) the environment wrapper, and (3) the decoding/interaction constraints as consistent as possible.

### E.1 Prompt-Based Baselines

Prompt-based methods do not update model parameters at inference time. To ensure fair comparison, we standardize the environment wrapper and constrain each method to output actions in the same executable format required by the benchmark. Unless otherwise specified, we follow each method’s original prompting protocol and include a small number of in-context demonstrations (few-shot examples) to induce the intended reasoning and interaction format.

*   •
    
    ReAct \[[81](#bib.bib52 "ReAct: Synergizing reasoning and acting in language models")\]: We follow the canonical Thought–Action–Observation loop, where the agent alternates between intermediate reasoning and an executable action at each step. The agent is prompted to produce an action string compatible with the environment API.
    
*   •
    
    Reflexion \[[65](#bib.bib54 "Reflexion: Language agents with verbal reinforcement learning")\]: Reflexion is inherently multi-trial: after each attempt, the agent generates a self-reflection based on episode feedback and stores it in memory to guide subsequent attempts. Accordingly, we report Reflexion under a pass@k protocol that matches its intended evaluation procedure, and we distinguish these results from pass@1 numbers.
    
*   •
    
    ADaPT \[[61](#bib.bib57 "ADaPT: As-needed decomposition and planning with language models")\]: We implement ADaPT as a prompt-based hierarchical agent with a planner and an executor. The planner proposes a sub-goal (or plan sketch) given the task and current observation, and the executor produces step-level actions conditioned on the planner’s output.
    

### E.2 Fine-Tuning-Based Baselines

Fine-tuning baselines update model parameters (via LoRA) using RL-style objectives while keeping the environment interface identical to other methods.

*   •
    
    GRPO \[[63](#bib.bib81 "DeepSeekMath: Pushing the limits of mathematical reasoning in open language models")\]: We use GRPO as a single-agent RL fine-tuning baseline. Concretely, a single policy is fine-tuned to maximize environment rewards without introducing an explicit hierarchy; the agent directly maps observations to actions under the same interaction budget used by other methods.
    
*   •
    
    Glider \[[17](#bib.bib100 "Divide and conquer: Grounding LLMs as efficient decision-making agents via offline hierarchical reinforcement learning")\]: Glider is a strong state-of-the-art hierarchical RL framework with a high-level planner and a low-level controller. However, unlike Multi2, it differentiates the two roles primarily through hierarchy prompts while sharing a single parameter-efficient adaptation module across both components, resulting in limited parameter-level decoupling between planning and execution. In addition, Glider trains both levels under a shared training paradigm, whereas Multi2 explicitly treats planning and execution as different optimization problems, using SFT for the planner and offline-to-online RL for the executor. Furthermore, Glider does not continue adapting the low-level executor online, which limits its ability to correct execution-side failures under environment feedback. These differences make Glider a strong and highly relevant comparison point for evaluating the benefits of role-specialized training and executor adaptation in Multi2.

#### Training Setting Analysis

We provide additional analysis of the online RL stage in terms of training cost, stability, and learning dynamics.

Relative to offline training, the online RL stage introduces only a modest computational overhead, adding approximately 20% additional training time. Despite this, it consistently improves performance from the offline-only setting (60.68 / 29.04) to the online-adapted setting (69.30 / 30.12) on the ScienceWorld ID / OOD splits. This indicates a favorable cost–performance trade-off, where a small increase in training time yields a substantial performance gain.
